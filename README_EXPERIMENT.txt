Unified Area + Segment experiment

- New work files no longer ask Compass vs Area.
- Every new file starts with Area 1 and Segment 1.
- New Segment creates the next numeric segment and selects it.
- New areas start fresh with Segment 1 instead of inheriting prior sides.
- Compass is optional: use + Compass Labels to add N/E/S/W to the current area.
- Existing compass-mode files remain supported by the old workspace logic.
- Existing storage fields (sideMode, assignedSide, sideTags) are intentionally retained for compatibility.
